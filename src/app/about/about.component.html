About Gazebo

Gazebo is a collection of open source software libraries designed to simplify development of high-performance applications. The primary audience for Gazebo are robot developers, designers, and educators. However, Gazebo has been structured to suit many different use cases. Each library within Gazebo has minimal dependencies, allowing them to be used in tasks ranging from solving mathematical transforms, to video encoding, and up to simulation and process management. Just choose the libraries you need for your application without committing to a whole ecosystem.

Trust and reliablity has been established through a curation and maintainence process lead by Open Robotics in collaboration with a community of developers. Each library within Gazebo has been designed to serve a specific purpose. This philosophy reduces code clutter, establishes consistency between libraries, and simplifies finding solutions. Development and maintenance adheres to a protocol consisting of multiple reviews, code checkers, and continuous integration. Take a look at Gazebo's roadmap for upcoming features and the latest improvements.

Guiding principles

The development of Gazebo follows these core principles.

  • Generality: Gazebo does not specialize in one type of robot. Features are meant to apply to as many use cases in robotics as possible.
  • Stability: Stable releases are provided to the community periodically. Subsequent improvements to stable releases strive to keep backwards compatibility of API, ABI and behaviour. Breaking changes target an upcoming release.
  • Updates: New backwards compatible features are added to stable releases whenever possible. This way, users can take advantage of new features without the need to upgrade to a later release, and can postpone upgrade efforts until breaking changes are needed.
  • Ease of use: Default behaviors should be beginner friendly. For example, by default, Gazebo starts running a graphical interface. Advanced users can run headless, but extra configuration is necessary to achieve this behavior.
  • Modularity: Gazebo packages are as independent as possible so that they can be used in a variety of projects besides Gazebo. For example, a user of Gazebo Rendering does not need to install, compile, or even know about Gazebo Physics.
  • Extensibility: Users do not need to fork the core libraries or compile from source to add functionality. For example, a new rendering engine can be loaded as a plugin.
  • Flexibility: Users are given options to opt in or out of features whenever reasonable. For example, physics is integrated into Gazebo as a plugin, so users can load a simulation without physics, or with alternative physics, if desired.
  • Maintainability: The long-term maintainability of the code is taken into account during development. To that end, the project follows strict testing, documentation, linting and style standards.
  • Portability: Gazebo is supported on all major operating systems: Linux, macOS and Windows. This impacts how dependencies are chosen and features are implemented.

History

Gazebo was started in 2002. After over 15 years of development it was time for a significant upgrade and modernization. This upgrade also provided the opportunity to move away from a monolithic architecture to a collection of loosely coupled libraries.

We branded the new development effort Igntion in order to differentiate it from Gazebo that had come before, which is now referred to as Gazebo Classic. Those changes have gone very well, thanks in no small part to contributions and support from our worldwide community of users and other stakeholders.

In 2022 we encountered a trademark obstacle regarding the use of the name "Ignition". We used this as an opportunity to switch back the widely known name "Gazebo". Going forward, the modern robotics software collection formerly known as Ignition, is now branded Gazebo.

Terminology

Gazebo

The complete Gazebo platform which includes software libraries and cloud services.

  • "Gazebo includes the Gazebo Simulator."
  • "There is a new release of Gazebo out called Garden."

Gazebo Classic

A version of Gazebo prior to development of latest Gazebo libraries. The classic versions are numerical with major version number ranging from 0 to 11.

  • "My project depends on Gazebo Classic."
